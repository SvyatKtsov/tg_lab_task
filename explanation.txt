dataset:  окремого датасету не збирається - використовуються претреновані моделі на публічних даних (COCO, відкриті ANPR датасети). 
Тренування з нуля зайняло б тиждень+

Моделі:
2 models:  for car detection,  and car license plate detection (OCR?), combine them
for color: if most pixels in a bbox have one color, it will be the color

car det: train or use ? можна натренувати але потрібно буде більше 3-5 днів. зібрати дані, обробити і тд. тиждень чи більше на це піде.


YOLOv8n - детекція авто, клас car (id=2)
fast-alpr (yolo-v9-t-384 + cct-xs-v2) - детекція та розпізнавання номерних знаків

алгоритм:
- читання відеопотоку (RTSP або файл) через OpenCV
- ресайз кадру до 640 на 640, детекція авто через YOLOv8n (onnx)
- фільтрація дрібних bbox (менше 50px) - шум/далекі авто
- SORT трекер - зіставляє детекції між кадрами через IoU + фільтр Калмана, присвоює кожному авто унікальний ID
- для кожного треку: обрізка crop -> усереднення пікселів центральної частини bbox -> колір рамки = реальний колір авто
- ALPR запускається раз на 5 кадрів і тільки якщо номер ще не закешований для цього ID -> результат зберігається в plate_cache[id]
- на фрейм виводимо рамку кольором авто, ID зверху, номер знизу, FPS у куті

В якості оптимізації, оскільки в нас не custom модель (на чистому наприклад PyTorch), використовуємо onnx експорт, без pruning й інших штучок.
YOLOv8n експортується в ONNX (один раз при першому запуску), далі інференс через ONNX Runtime без PyTorch; imgsz=320 для детекції - менше роздільна здатність, 
швидший інференс;  також пропускаємо малі bbox`и
і використовуємо кешування номерів - OCR не виконується для вже розпізнаного авто; ALPR викликається раз на 5 кадрів, а не кожний кадр.
